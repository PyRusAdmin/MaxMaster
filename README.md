# Max — программа для парсинга пользователей OneMe API

Программа для массового поиска пользователей в OneMe/Max по номерам телефонов с сохранением результатов в Excel и использованием SQLite базы данных для управления очередью.

## Возможности

- 🔍 Поиск пользователей по номеру телефона
- 💾 Экспорт данных в Excel (автоматически после каждого найденного пользователя)
- 📊 Очередь номеров на основе SQLite (сохранение прогресса при остановке)
- 🔄 Автоматическое переподключение при разрыве соединения
- ⏱️ Гибкая настройка задержек (между запросами и при rate limit)
- 🎨 Красивый консольный интерфейс на Rich
- 📝 Логирование всех событий

## Требования

- **Python 3.10+** (требуется поддержка `match/case` и современных типов)
- **Windows/Linux/macOS**

## Настройка

### 1. Файл `.env`

Создайте файл `.env` в папке с программой и настройте параметры:

```env
# Номер телефона для авторизации в Max (без +)
PHONE_NUMBER=79991234567

# Задержка между запросами к API (секунды)
SLEEP_BETWEEN_REQUESTS=3

# Задержка при получении rate limit (секунды)
SLEEP_ON_RATELIMIT=30

# Путь к базе данных SQLite
DB_PATH=data/queue.db

# Путь к файлу с номерами телефонов
NUMBERS_FILE=input/numbers.txt

# Путь к файлу результатов Excel
EXCEL_FILE=output/users.xlsx
```

**Описание параметров:**

| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| `PHONE_NUMBER` | — | Номер телефона в международном формате (без `+`) |
| `SLEEP_BETWEEN_REQUESTS` | `5` | Задержка между запросами к API (рекомендуется 2-5 сек) |
| `SLEEP_ON_RATELIMIT` | `30` | Задержка при получении ошибки rate limit |
| `DB_PATH` | `data/queue.db` | Путь к SQLite базе данных очереди |
| `NUMBERS_FILE` | `input/numbers.txt` | Файл со списком номеров для поиска |
| `EXCEL_FILE` | `output/users.xlsx` | Файл для сохранения результатов |

### 2. Файл с номерами `input/numbers.txt`

Создайте файл `input/numbers.txt` и запишите в него номера телефонов (каждый с новой строки):

```
79991234567
79991234568
79991234569
```

### Описание пунктов меню:

1. **▶ Продолжить / начать** — запуск обработки очереди номеров

3. **📊 Статистика** — подробная информация об очереди и настройках
4. **❌ Выйти** — завершение работы программы

### Особенности работы:

- ✅ Прогресс сохраняется в БД — можно остановить в любой момент (`Ctrl+C`) и продолжить позже

- ✅ Ошибки и rate limit обрабатываются автоматически
- ✅ При разрыве соединения происходит автоматическое переподключение

## База данных

Программа использует SQLite базу данных (`data/queue.db`) для управления очередью номеров.

### Таблица `phone_queue`:

| Поле | Тип | Описание |
|------|-----|----------|
| `phone` | VARCHAR | Номер телефона (уникальный) |
| `added_at` | DATETIME | Дата добавления в очередь |

### Преимущества использования БД:

- 🔹 **Сохранение прогресса** — можно остановить программу в любой момент
- 🔹 **Защита от дубликатов** — номера не повторяются благодаря `UNIQUE` ограничению
- 🔹 **Пакетная обработка** — номера загружаются из файла один раз
- 🔹 **Гибкость** — можно добавлять номера в очередь на лету

## Формат результатов Excel

Результаты сохраняются в файл `output/users.xlsx` со следующими колонками:

| Колонка | Описание |
|---------|----------|
| ID пользователя | Уникальный идентификатор в OneMe |
| Имя | Имя пользователя (может быть несколько через ` \| `) |
| Искомый телефон | Номер, по которому искали |
| Фото (URL) | Ссылка на основное фото |
| Фото (raw URL) | Ссылка на фото в исходном качестве |
| Пол | Пол пользователя |
| Описание | Биография/описание профиля |
| Статус аккаунта | Статус аккаунта (числовое значение) |
| ID фото | Идентификатор фотографии |
| Последняя активность | Дата и время последнего обновления |
| Платформы | Список платформ (ONEME, TT и др.) |
| Ошибка | Текст ошибки (если поиск не удался) |

## Логирование

Все события записываются в `logs/log.log`:

- ✅ Успешные поиски
- ❌ Ошибки и исключения
- ⚠️ Rate limit и переподключения
- 📊 Статистика обработки

Размер лог-файла автоматически ограничивается (rotation at 1 MB).

## Обработка ошибок

| Ситуация | Действие |
|----------|----------|
| **Rate limit** | Пауза `SLEEP_ON_RATELIMIT` сек, повторная попытка |
| **Разрыв соединения** | Автоматическое переподключение (до 3 попыток) |
| **Пользователь не найден** | Запись в лог, номер удаляется из очереди |
| **Ошибка API** | Запись ошибки в Excel, номер удаляется из очереди |

## Примеры использования

### Быстрый старт

1. Получите номер на https://365sms.biz/activations
2. Зарегистрируйтесь в Max/OneMe
3. Пропишите номер в `.env`
4. Добавьте номера для поиска в `input/numbers.txt`
5. Запустите `Запуск.bat`
6. Выберите пункт `[1]` для начала обработки
7. Результаты появятся в `output/users.xlsx`

### Массовый парсинг

Для обработки большой базы номеров (1000+):

1. Установите `SLEEP_BETWEEN_REQUESTS=3` (баланс скорости и безопасности)
2. Установите `SLEEP_ON_RATELIMIT=60` (при частых блокировках)
3. Загрузите номера через пункт `[2]`
4. Запустите обработку `[1]`
5. При необходимости останавливайте `Ctrl+C` — прогресс сохранится
